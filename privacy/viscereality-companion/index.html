Viscereality Companion Privacy Policy

This policy explains how Viscereality Companion handles data during an attended provisioning session with a user-controlled Meta Quest.

Last updated: July 15, 2026

Scope

This privacy policy applies to Viscereality Companion, an Android application published by Mesmer Prism.

Data the app accesses

Viscereality Companion accesses only the data needed for a user-requested provisioning session:

  • information exposed by an attached Meta Quest USB ADB interface;
  • the Quest's local-network address;
  • an ADB public/private key pair generated by the app;
  • APK files the user explicitly selects, including package name, version, file size, SHA-256 digest, and signing-certificate digest; and
  • a local receipt describing packages and permissions verified on the Quest.

The app does not request access to contacts, location, photos, microphone, camera, advertising identifiers, or the phone's installed-app inventory.

How data is used and transferred

The app uses this information only to authenticate an attended ADB session, enable local Wi-Fi ADB, send user-selected APKs directly to a Quest controlled by the user, verify their installation, and record the result locally.

Selected APKs, ADB authentication messages, and provisioning commands are transferred directly between the phone and the user's Quest over USB or the local network. The app does not transmit this information to Mesmer Prism or to a developer-operated server. It contains no advertising or analytics SDK. Mesmer Prism does not sell or share data handled by the app.

Storage, retention, and deletion

The ADB key, APK working copies, and provisioning receipts are stored in the app's private storage on the phone. Android cloud backup and device-transfer backup are disabled for this data. The data remains until the user clears the app's storage or uninstalls the app.

Clearing storage or uninstalling Viscereality Companion deletes the phone-side ADB key, APK working copies, and receipts. Software already installed on the Quest must be removed separately through the Quest or its authorized developer tools. The app does not create a user account and Mesmer Prism retains no server-side copy to delete.

Security

The app requires an Android USB-access decision and the Quest's visible ADB authorization decision. It accepts only fixed provisioning package identities and approved signing certificates, requires the Quest connection to report Android shell authority, and verifies installation and permission state before writing a receipt.

Children

Viscereality Companion is a developer tool and is not directed to children.

Changes and contact

Material changes to this policy will be published with a new last-updated date. Privacy questions can be sent to mesmerprism@gmail.com.
